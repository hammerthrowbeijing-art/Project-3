# Break 50 P3 Repository

This repository satisfies the course requirement for a public GitHub submission: it includes the project plan, reproducible analysis code, documentation, and all analytic outputs. The raw dataset — public X (Twitter) comments collected around Donald Trump's July 2024 appearance on Bryson DeChambeau's *Break 50* YouTube series — is kept outside version control per course policy. The pipeline reads it at runtime from a path you supply via an environment variable, so nothing about the dataset's local layout is hardcoded.

## Research Question

The revised project asks how the Break 50 comments organize Trump's appearance as **contested discourse**:

- Which comments frame the event as political, sport-centered, or a blend of both?
- How are stance positions distributed across supportive, oppositional, depoliticizing, ambiguous, and sport-centered comments?
- Which discourse positions receive more visible reaction in likes, retweets, and comment views?

`Author's attention quantity` is still used in the pipeline, but now as a secondary control rather than as the main theoretical construct.

## How To Reproduce

The pipeline expects a local copy of the Break 50 comment dataset placed anywhere on your machine. Point the script at it by setting the `BREAK50_DATA_PATH` environment variable, then run the analysis script. The figures and markdown tables in `outputs/` will regenerate from the raw file.

```powershell
# Windows PowerShell
$env:BREAK50_DATA_PATH="<path to your local copy>"
py -X utf8 scripts/run_analysis.py
```

```bash
# macOS / Linux
export BREAK50_DATA_PATH="<path to your local copy>"
python scripts/run_analysis.py
```

Python dependencies are listed in [`requirements.txt`](./requirements.txt). If `BREAK50_DATA_PATH` is unset, the script falls back to `./data/` (a gitignored local folder) and will raise a clear error if the file is not found there.

## Notes For GitHub Submission

- Do not upload `xlsx`, `csv`, or other raw data extracts — the included `.gitignore` guards against accidental commits.
- Commit only code, markdown documentation, figures, and meaningful analysis outputs.
- If group members continue revising the analysis, rerun `scripts/run_analysis.py` so that figures, markdown tables, and the analysis summary stay synchronized with the report.
